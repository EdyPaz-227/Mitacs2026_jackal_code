CONTEXTO PARA CONTINUAR — IMPLEMENTACIÓN JACKAL ROS 2 CON CONTROL PS

Objetivo
Adaptar el paquete ROS 2 de trayectoria para un Jackal J100 real. El robot debe poder manejarse manualmente y ejecutar una trayectoria automática únicamente al presionar X en el control de PlayStation.

Robot y sistema
- Robot: Clearpath Jackal J100
- Hostname: cpr-j100-0751
- Usuario/IP SSH: isami@10.160.224.41
- Ubuntu: 22.04.5 LTS
- ROS: ROS 2 Humble
- Arquitectura: x86_64
- Workspace: ~/jackal_ws
- Paquete: jackal_trajectory_control

Topics reales
- /j100_0751/cmd_vel
- /j100_0751/joy_teleop/cmd_vel
- /j100_0751/joy_teleop/joy
- /j100_0751/platform/cmd_vel_unstamped
- /j100_0751/platform/odom
- /j100_0751/platform/odom/filtered
- /j100_0751/platform/emergency_stop
- /j100_0751/rc_teleop/cmd_vel
- /j100_0751/sensors/lidar3d_0/scan

Tipos comprobados
- /j100_0751/cmd_vel: geometry_msgs/msg/Twist
- /j100_0751/joy_teleop/cmd_vel: geometry_msgs/msg/Twist
- /j100_0751/platform/cmd_vel_unstamped: geometry_msgs/msg/Twist
- /j100_0751/platform/odom/filtered: nav_msgs/msg/Odometry

Diseño solicitado
- El paquete inicia manualmente por ahora; más adelante se configurará autoarranque.
- Al iniciar, el robot debe quedar quieto en IDLE.
- La teleoperación manual debe seguir funcionando.
- X inicia la trayectoria seleccionada.
- Círculo cambia el modo: horizontal → vertical → inverso.
- Cuadrado guarda la pose actual como home local (0,0,0), sin reiniciar /odom.
- Triángulo:
  - pulsación corta: cancelación inmediata;
  - mantener presionado: regreso controlado al home.
- L1/R1 tienen prioridad manual y cancelan la trayectoria automática.
- Si X se presiona mientras ya corre una trayectoria, se ignora.
- Todas las trayectorias deben regresar por los mismos puntos en orden inverso.

Índices reales del control
- X = buttons[0]
- Círculo = buttons[1]
- Cuadrado = buttons[2]
- Triángulo = buttons[3]
- L1 = buttons[9]
- R1 = buttons[10]

ros2 launch jackal_trajectory_control joystick_trajectory.launch.py \
  columns:=2 \
  rows:=2 \
  spacing_x:=0.50 \
  spacing_y:=0.50 \
  max_linear_speed:=0.10 \
  max_angular_speed:=0.25 \

Resultado de launch
El nodo inicia correctamente y muestra:
- Velocity input: cmd_vel
- Odometry: platform/odom/filtered
- Joystick: joy_teleop/joy
- Buttons: X=start, Circle=change mode, Square=set home, Triangle=cancel/hold for home, L1/R1=manual override
- “Press Square first to define home.”

Problema actual
Aunque el launch inicia, al presionar Cuadrado o X no aparece reacción y el robot no se mueve.

El topic del joystick mostraba Publisher count: 1, pero ros2 topic echo quedaba esperando y no recibía mensajes. Se descubrió que era necesario ejecutar manualmente en otra terminal:

ros2 run joy_linux joy_linux_node --ros-args \
  -r joy:=/j100_0751/joy_teleop/joy

Después de ejecutar ese nodo, este comando sí comenzó a mostrar los botones:

ros2 topic echo /j100_0751/joy_teleop/joy

La captura confirmó que Cuadrado pone buttons[2] = 1. Sin embargo, el controlador de trayectoria todavía no reacciona y el Jackal sigue sin moverse.

Comandos de entorno usados
source /etc/clearpath/setup.bash
source ~/jackal_ws/install/setup.bash

Siguiente diagnóstico recomendado
1. Confirmar que el controlador está suscrito al topic correcto:
   ros2 node info /j100_0751/joystick_trajectory_controller

2. Revisar publishers, subscribers y QoS:
   ros2 topic info -v /j100_0751/joy_teleop/joy

3. Verificar que el nodo recibe odometría:
   ros2 topic hz /j100_0751/platform/odom/filtered

4. Ver el log del controlador mientras se presiona Cuadrado. Debe aparecer un mensaje de home establecido. Si no aparece, revisar la suscripción de Joy.

5. Sospecha principal:
   Incompatibilidad de QoS entre joy_linux_node y la suscripción del controlador. El controlador probablemente usa QoS por defecto RELIABLE, mientras el joystick puede publicar BEST_EFFORT. Cambiar la suscripción a sensor_data QoS o BEST_EFFORT:
   qos_profile_sensor_data
   o
   QoSProfile(reliability=ReliabilityPolicy.BEST_EFFORT, depth=10)

6. También revisar si el topic relativo se resuelve realmente a:
   /j100_0751/joy_teleop/joy

7. Tras corregir Joy, probar la secuencia:
   - lanzar el paquete;
   - presionar Cuadrado;
   - confirmar log “home set”;
   - presionar X;
   - confirmar cambio a RUNNING.

8. Si el controlador entra a RUNNING pero no se mueve, verificar la salida:
   ros2 topic echo /j100_0751/cmd_vel

9. Si /j100_0751/cmd_vel publica velocidades pero el robot no se mueve, revisar el multiplexor/prioridad entre:
   - /j100_0751/cmd_vel
   - /j100_0751/joy_teleop/cmd_vel
   - /j100_0751/platform/cmd_vel_unstamped

Seguridad
- Hacer la primera prueba con grilla 2×2, spacing 0.5 m y 0.10 m/s.
- Espacio despejado.
- Tener acceso inmediato al paro físico.
- No usar todavía la grilla completa 8×5.
